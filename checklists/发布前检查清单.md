# 发布前检查清单

这个清单用于发布 GitHub 仓库前做最后检查。

## 1. 基础文件

- [ ] README.md 存在
- [ ] LICENSE 存在
- [ ] .gitignore 存在
- [ ] .env.example 存在
- [ ] CONTRIBUTING.md 存在
- [ ] CHANGELOG.md 存在

## 2. 敏感信息

- [ ] 没有提交真实 `.env`
- [ ] 没有提交真实 API Key
- [ ] 没有提交数据库密码
- [ ] 没有提交真实客户资料
- [ ] 示例文档不包含敏感信息

## 3. Python 检查

执行：

```bash
python -m compileall examples apps
```

确认：

- [ ] Python 文件可以编译
- [ ] 没有语法错误

## 4. 前端检查

每个前端项目执行：

```bash
npm install
npm run build
```

需要检查：

- [ ] ai-chat-assistant 前端 build 通过
- [ ] resume-ai-assistant 前端 build 通过
- [ ] personal-rag-knowledge-base 前端 build 通过
- [ ] enterprise-ai-customer-service 前端 build 通过

## 5. Docker 检查

至少选择一个项目执行：

```bash
docker compose up --build
```

建议检查：

- [ ] ai-chat-assistant Docker 能启动
- [ ] enterprise-ai-customer-service Docker 能启动

## 6. 文档检查

- [ ] 新手入门指南可读
- [ ] 学习路线清晰
- [ ] 完整课程目录链接正确
- [ ] apps 目录说明清晰
- [ ] troubleshooting 存在
- [ ] 当前版本说明存在

## 7. 项目检查

- [ ] 4 个 apps 项目都有 README
- [ ] 4 个 apps 项目都有启动方式
- [ ] 4 个 apps 项目都有 Dockerfile 或 docker-compose
- [ ] 示例文档可以上传测试

## 8. GitHub 展示

- [ ] README 首屏能说明仓库价值
- [ ] 仓库定位明确
- [ ] 明确说明当前是教学版
- [ ] 截图目录已规划
- [ ] 后续升级路线清晰

## 9. 发布后建议

发布后可以继续补：

- [ ] 项目截图
- [ ] 生产级 pro 版本
- [ ] 更详细章节讲义
- [ ] 视频脚本
- [ ] 单元测试
