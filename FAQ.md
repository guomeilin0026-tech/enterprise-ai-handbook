# FAQ：常见问题

## 1. 我是小白，可以学这套课程吗？

可以。

这套课程不是从训练大模型开始，而是从企业 AI 应用开发开始。

你会按这个顺序学习：

```text
大模型基础
-> Prompt
-> FastAPI 后端
-> Vue3 前端
-> RAG 知识库
-> Agent 工具调用
-> Docker 部署
```

## 2. 我需要先学算法和数学吗？

不需要。

本仓库目标是：

```text
企业 AI 应用全栈工程师
```

不是：

```text
大模型算法工程师
```

所以你先学会怎么用大模型做产品，再逐步补算法。

## 3. 我需要会 Python 吗？

需要一点，但不需要一开始很强。

你只需要先掌握：

- 变量
- 列表
- 字典
- 函数
- 模块
- FastAPI 接口

## 4. 我会 Vue3，有优势吗？

有。

企业 AI 应用需要前端页面，例如：

- 聊天界面
- 知识库管理页
- 文档上传页
- Agent 工具调用过程
- 用量统计页

会 Vue3 可以让你更快做出完整产品。

## 5. 为什么 API Key 不能放前端？

因为前端代码会被用户看到。

如果 API Key 放前端，别人可以盗用你的模型额度。

正确做法：

```text
Vue3 前端 -> FastAPI 后端 -> 大模型 API
```

API Key 只放后端 `.env`。

## 6. 为什么项目默认有 mock LLM？

因为小白第一步最重要的是跑通流程。

如果一开始就要求真实 API Key，很多人会卡在账号、额度、网络和模型配置上。

mock LLM 的作用是：

```text
先理解架构，再接真实模型。
```

## 7. 这些项目能写进简历吗？

可以，但建议你至少完成最终项目：

```text
企业知识库 + 智能客服系统
```

并能讲清楚：

- RAG 流程
- Agent 工具调用
- 权限隔离
- Token 用量统计
- Docker 部署

## 8. 学完就是企业 AI 全栈工程师了吗？

学完并做完项目后，你具备企业 AI 应用全栈的入门到中级能力。

真正达到可上岗，还要能：

- 独立排查问题
- 讲清楚架构
- 部署上线
- 理解安全和成本
- 根据业务改造项目

## 9. RAG 和 Agent 哪个更重要？

都重要，但学习顺序建议：

```text
先学 RAG，再学 Agent。
```

RAG 解决“查资料再回答”。

Agent 解决“调用工具完成任务”。

## 10. 我应该先看课程还是先跑项目？

建议：

```text
先看新手入门指南
再按学习路线学
每学一段就跑对应项目
```

不要只看文档，也不要完全不看概念就直接跑项目。

## 11. 项目启动失败怎么办？

先看：

- [troubleshooting/常见报错.md](./troubleshooting/常见报错.md)

再检查：

- 是否在正确目录
- 是否安装依赖
- 是否复制 `.env.example` 为 `.env`
- 端口是否被占用
- 后端是否启动

## 12. 教学版和生产版有什么区别？

教学版为了让你快速理解，使用：

- 本地 JSON
- mock LLM
- 简单向量检索
- 模拟 Agent

生产版需要升级：

- PostgreSQL
- Qdrant
- 真实大模型 API
- JWT 鉴权
- Redis 队列
- Nginx
- 日志和监控

具体看：

- [docs/生产级升级路线.md](./docs/生产级升级路线.md)
