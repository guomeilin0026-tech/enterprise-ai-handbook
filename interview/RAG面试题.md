# RAG 面试题

## 1. RAG 是什么？

RAG 是“先检索资料，再让大模型根据资料回答”的技术方案，常用于企业知识库问答。

## 2. RAG 的完整流程是什么？

```text
上传文档
-> 解析文档
-> 文本切分
-> 生成 Embedding
-> 存入向量数据库
-> 用户提问
-> 检索相关片段
-> 拼接 Prompt
-> 大模型回答
-> 展示引用来源
```

## 3. 为什么文档要切分？

因为长文档无法全部放进模型上下文，切分后检索更精准，成本更低，也方便展示引用来源。

## 4. Embedding 是什么？

Embedding 是把文本转换成向量，用于计算语义相似度。

## 5. 如何减少 RAG 幻觉？

- Prompt 中要求只根据资料回答
- 资料不足时允许回答不知道
- 展示引用来源
- 设置相似度阈值
- 对高风险答案做人工审核
