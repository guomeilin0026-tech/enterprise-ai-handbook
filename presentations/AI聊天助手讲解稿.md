# AI 聊天助手讲解稿

## 30 秒介绍

这个项目是一个基于 Vue3 和 FastAPI 的 AI 聊天助手，用来跑通大模型应用的最小闭环。前端负责聊天界面和流式展示，后端负责接收问题、保护 API Key，并调用大模型接口。

## 项目解决什么问题

它解决的是：

```text
如何从 0 搭建一个大模型聊天应用。
```

学习重点不是聊天本身，而是理解：

- API Key 为什么必须放后端
- 前端如何请求后端
- 后端如何调用模型
- 流式输出如何提升体验

## 技术栈

```text
前端：Vue3 + Vite
后端：FastAPI
模型：OpenAI-compatible API / Mock LLM
部署：Docker Compose
```

## 核心流程

```text
用户输入问题
-> Vue3 调用 /api/chat/stream
-> FastAPI 接收问题
-> 后端调用模型或 mock 回复
-> 后端流式返回
-> 前端逐段显示
```

## 面试讲法

如果面试官问“你这个项目的重点是什么”，可以这样回答：

```text
这个项目重点是大模型应用的基础架构。前端不直接调用模型，所有模型调用都通过 FastAPI 后端完成，避免 API Key 暴露。同时我实现了流式输出，让用户可以像 ChatGPT 一样逐步看到回复。
```

## 可改进点

- 增加用户登录
- 保存聊天记录到数据库
- 支持多模型配置
- 增加 Token 用量统计
