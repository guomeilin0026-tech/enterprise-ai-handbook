# 课程设计标准

本课程的目标不是让学习者“看过很多概念”，而是让学习者真正做到：

```text
能理解
能写代码
能做项目
能讲清楚
能部署上线
```

## 每章必须回答的问题

每章内容都要让学习者知道：

- 这个概念是什么
- 为什么企业项目需要它
- 它解决什么问题
- 它在最终项目里出现在哪里
- 它怎么用代码实现
- 它有哪些常见坑
- 面试时怎么讲

## 每章推荐结构

```text
本章目标
小白解释
核心概念
企业场景
技术流程
最小代码示例
工程化注意点
常见坑
实战任务
作业
面试题
本章总结
```

## 内容难度要求

同一个知识点要分三层讲：

```text
第一层：一句话解释
第二层：流程解释
第三层：工程落地
```

## 代码示例要求

代码示例必须：

- 最小可运行
- 有 README
- 有启动命令
- 不依赖真实密钥才能理解
- 涉及密钥时使用 `.env.example`

## 项目要求

每个项目必须包含：

- 项目背景
- 用户故事
- 功能清单
- 页面设计
- API 设计
- 数据库设计
- 验收标准
- 简历描述
