# 第 9 章：RAG 参考答案

## 1. RAG 是什么

参考答案：

```text
RAG 是先从知识库检索相关资料，再把资料和用户问题一起交给大模型，让模型基于资料回答。它常用于企业知识库问答。
```

## 2. 为什么文档要切分

参考答案：

```text
因为长文档无法全部放进模型上下文，成本高且检索不精准。切分后可以只检索最相关片段，降低成本，提高回答准确性。
```

## 3. Embedding 的作用

参考答案：

```text
Embedding 是把文本转换成向量，用来计算语义相似度。这样用户问法和文档原文不完全一样时，系统也能找到意思相近的内容。
```

## 4. 如何减少 RAG 幻觉

参考答案：

```text
可以通过限制模型只根据资料回答、资料不足时拒答、展示引用来源、设置相似度阈值、避免塞入无关片段、关键场景人工审核来减少幻觉。
```

## 5. 企业 RAG 如何做权限隔离

参考答案：

```text
文档、知识库、文档片段和向量 payload 都要保存 user_id 或 tenant_id。检索向量时必须带过滤条件，后端也要统一校验当前用户是否有权限访问该知识库。
```

## 6. RAG 流程图文字版

```text
上传文档
-> 解析文档
-> 文本切分
-> 生成 Embedding
-> 存入向量数据库
-> 用户提问
-> 问题生成 Embedding
-> 检索相关片段
-> 拼接 Prompt
-> 大模型回答
-> 展示引用来源
```
