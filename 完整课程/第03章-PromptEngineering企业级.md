# 第 3 章：Prompt Engineering（企业级）

## 3.1 Prompt 是什么

Prompt 是你给大模型的任务说明。

差的 Prompt：

```text
帮我写一下
```

好的 Prompt：

```text
你是一名资深 HR。
请根据候选人简历和岗位 JD，输出岗位匹配度、优势、不足和修改建议。
输出必须使用 JSON。
```

## 3.2 企业 Prompt 的核心结构

推荐结构：

```text
角色：你是谁
任务：你要做什么
输入：用户会给什么
规则：不能做什么
输出：必须按什么格式返回
兜底：资料不足时怎么回答
```

## 3.3 角色设定

角色设定让模型进入任务场景。

例子：

```text
你是一个企业客服知识库助手。
你的职责是根据提供的资料回答员工问题。
```

## 3.4 任务拆解

不要让模型一次性“随便发挥”，要拆步骤。

例子：

```text
请按步骤完成：
1. 先理解用户问题
2. 再判断资料是否包含答案
3. 如果包含，基于资料回答
4. 如果不包含，回答“资料中没有找到相关信息”
```

## 3.5 Few-shot 示例

Few-shot 就是给模型示例。

```text
示例输入：员工问年假怎么算？
示例输出：根据员工手册第 3 条，年假按入职年限计算...
```

模型会模仿示例格式。

## 3.6 企业级 Prompt 模板

### 知识库问答模板

```text
你是企业知识库助手。
请只根据【资料】回答【问题】。
如果资料中没有答案，请回答：资料中没有找到相关信息。
不要编造资料中不存在的内容。

【资料】
{context}

【问题】
{question}
```

### 简历分析模板

```text
你是资深 HR。
请根据【简历】和【岗位 JD】分析候选人匹配度。
请输出：匹配度、优势、不足、修改建议。
```

### 数据分析模板

```text
你是业务数据分析师。
请根据数据摘要输出趋势、异常点、原因猜测和行动建议。
不确定的结论请标注为“推测”。
```
