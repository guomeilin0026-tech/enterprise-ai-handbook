# 第 5 章：Structured Output

## 5.1 什么是结构化输出

结构化输出就是让 AI 按固定格式返回数据。

普通回答：

```text
这个简历还不错，匹配度大概 80 分...
```

结构化输出：

```json
{
  "score": 80,
  "strengths": ["Vue3 经验", "项目经历完整"],
  "weaknesses": ["后端经验不足"],
  "suggestions": ["补充 FastAPI 项目经验"]
}
```

## 5.2 为什么企业项目需要结构化输出

因为前端需要稳定渲染。

结构化输出适合：

- 表单抽取
- 简历分析
- 合同字段提取
- 发票识别
- 工单分类
- 风险等级判断

## 5.3 Schema 思维

Schema 就是字段规则。

例如简历分析结果：

```text
score：数字
strengths：字符串数组
weaknesses：字符串数组
suggestions：字符串数组
summary：字符串
```

## 5.4 后端校验

AI 输出不一定永远正确，所以后端要校验。

常见处理：

- JSON 解析失败，重试
- 缺字段，重试
- 类型错误，提示失败
- 内容为空，返回友好提示

## 5.5 前端渲染方式

结构化数据可以渲染成：

- 评分卡片
- 标签
- 列表
- 表格
- 风险等级
- 时间线
