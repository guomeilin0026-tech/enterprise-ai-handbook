# 第 11 章：企业 AI 工程化

## 11.1 Demo 和企业项目的区别

Demo 只要能跑。

企业项目需要：

- 稳定
- 安全
- 可维护
- 可监控
- 可扩展
- 可追踪问题

## 11.2 日志

必须记录：

- 用户请求
- 模型调用
- 工具调用
- 错误信息
- Token 用量
- 响应时间

但日志不能记录敏感明文，例如密码、API Key、身份证号。

## 11.3 异常处理

常见异常：

- 模型接口失败
- 网络超时
- JSON 解析失败
- 向量库不可用
- 文件解析失败
- 用户权限不足

企业系统不能直接崩溃，要返回友好提示。

## 11.4 重试

适合重试：

- 临时网络失败
- 模型接口偶发失败
- 结构化输出格式错误

不适合盲目重试：

- 参数错误
- 权限错误
- 文件格式不支持

## 11.5 限流和成本控制

大模型调用花钱，所以要控制：

- 每人每日次数
- 每人每日 Token
- 单次最大上下文
- 上传文档大小
- 并发请求数

## 11.6 队列

文档解析和向量化可能很慢，适合异步队列。

流程：

```text
用户上传文档
-> 创建任务
-> 后台解析和向量化
-> 前端查看任务状态
```
