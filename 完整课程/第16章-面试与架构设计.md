# 第 16 章：面试与架构设计

## 16.1 面试时你要讲清楚什么

你不能只说：

```text
我做了一个 AI 项目
```

你要讲：

- 解决什么问题
- 用了什么技术
- 架构怎么设计
- 用户提问后系统怎么处理
- 如何减少幻觉
- 如何保护数据
- 如何控制成本
- 如何部署上线

## 16.2 企业知识库架构

```text
Vue3 前端
-> FastAPI 后端
-> PostgreSQL 保存用户、文档、聊天记录
-> Qdrant 保存向量
-> 大模型 API 生成回答
-> Docker 部署
```

## 16.3 RAG 流程面试回答

可以这样讲：

```text
用户上传文档后，系统会解析文档内容，把文本切分成片段，对每个片段生成 Embedding，并存入向量数据库。
用户提问时，系统先把问题转成向量，到向量库中检索最相关的文档片段，再把这些片段和用户问题一起发给大模型。
模型根据检索到的资料生成回答，并在前端展示引用来源。
```

## 16.4 Agent 流程面试回答

```text
Agent 和普通聊天不同，它可以根据用户意图选择工具。
比如用户要查询订单状态，模型会返回工具名和参数，后端校验权限后执行订单查询工具，再把工具结果返回给模型，由模型组织成自然语言回复。
所有工具调用都会记录日志，高风险操作需要人工确认。
```

## 16.5 常见面试题

### 为什么 API Key 不能放前端？

因为前端代码会暴露给用户，API Key 放前端会被盗用，造成费用和安全风险。

### RAG 为什么能减少幻觉？

因为模型不是凭空回答，而是基于检索到的企业资料回答，并展示引用来源。

### 文档为什么要切分？

因为长文档无法全部放进上下文，切分后检索更精准，成本也更低。

### 如何做用户数据隔离？

每个知识库、文档、向量片段都绑定用户或组织 ID，检索时必须带权限过滤。

### 如何控制 Token 成本？

限制上下文长度、限制调用次数、统计用量、缓存高频问题、选择合适模型。

## 16.6 项目介绍模板

```text
我做的是一个企业知识库智能客服系统。
前端使用 Vue3，后端使用 FastAPI，数据库使用 PostgreSQL，向量数据库使用 Qdrant。
系统支持文档上传、PDF 解析、文本切分、Embedding 入库、语义检索、RAG 问答、引用来源、聊天记录、用户权限、Token 用量统计和 Docker 部署。
```
