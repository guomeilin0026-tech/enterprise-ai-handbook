# 数据库设计规范

## 1. 核心表

## users

用户表。

字段建议：

```text
id
username
email
password_hash
role
created_at
updated_at
```

## knowledge_bases

知识库表。

```text
id
user_id
name
description
created_at
updated_at
```

## documents

文档表。

```text
id
knowledge_base_id
user_id
filename
file_type
file_size
status
created_at
updated_at
```

## document_chunks

文档片段元数据表。

```text
id
document_id
knowledge_base_id
user_id
chunk_index
content
vector_id
created_at
```

## conversations

会话表。

```text
id
user_id
title
created_at
updated_at
```

## messages

消息表。

```text
id
conversation_id
user_id
role
content
sources_json
created_at
```

## model_usage_logs

模型用量表。

```text
id
user_id
model
input_tokens
output_tokens
total_tokens
cost
created_at
```

## tool_call_logs

工具调用日志表。

```text
id
user_id
tool_name
arguments_json
result_json
status
created_at
```

## 2. 设计原则

- 所有用户数据必须带 `user_id`
- 多租户系统还要加 `tenant_id`
- 重要表要有 `created_at` 和 `updated_at`
- 不保存明文密码
- 大字段和结构化结果可以使用 JSON 字段

## 3. 向量数据和业务数据关系

Qdrant 中的 payload 至少包含：

```json
{
  "user_id": "用户 ID",
  "knowledge_base_id": "知识库 ID",
  "document_id": "文档 ID",
  "chunk_id": "片段 ID",
  "content": "片段原文"
}
```

检索时必须带权限过滤条件。
