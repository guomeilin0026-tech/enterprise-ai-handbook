# 技术选型说明

## 1. 前端：Vue3 + TypeScript

选择原因：

- 学习者已有 Vue3 基础
- 适合做后台管理和聊天界面
- 组件生态成熟

推荐组合：

```text
Vue3 + TypeScript + Pinia + Vue Router + Element Plus
```

## 2. 后端：Python FastAPI

选择原因：

- AI 生态友好
- 上手快
- 自动生成接口文档
- 适合构建大模型应用 API

## 3. 业务数据库：PostgreSQL

保存：

- 用户
- 知识库
- 文档
- 聊天记录
- 用量日志
- 反馈记录

## 4. 向量数据库：Qdrant

保存：

- 文档片段向量
- 文档片段 payload
- 知识库 ID
- 用户或租户 ID

## 5. 大模型接口

推荐封装成 OpenAI-compatible 风格。

好处：

- 可以切换不同模型供应商
- 后端调用方式统一
- 便于做模型配置

## 6. 部署：Docker + Nginx

服务拆分：

- frontend
- backend
- postgres
- qdrant
- nginx

## 7. 不建议一开始使用的技术

- Kubernetes：对入门阶段太重
- 自训练大模型：成本和难度太高
- 复杂微服务：项目早期不需要
- 自研向量库：没有必要
