# 系统架构说明

目标系统：企业知识库 + 智能客服系统。

## 1. 总体架构

```text
Vue3 前端
  -> FastAPI 后端
  -> PostgreSQL 业务数据库
  -> Qdrant 向量数据库
  -> 大模型 API
  -> Docker / Nginx 部署
```

## 2. 核心模块

### 前端模块

- 登录页
- AI 聊天页
- 知识库管理页
- 文档管理页
- 模型配置页
- 用量统计页
- 用户反馈页

### 后端模块

- 用户认证模块
- 聊天模块
- 大模型模块
- Prompt 模板模块
- 文档解析模块
- RAG 检索模块
- Agent 工具模块
- 用量统计模块
- 日志模块

### 数据模块

- PostgreSQL：用户、知识库、文档、聊天记录、用量日志
- Qdrant：文档片段向量和检索 payload
- 文件存储：上传的原始文档

## 3. 用户提问流程

```text
用户输入问题
-> 前端发送到 FastAPI
-> 后端校验用户身份
-> 根据知识库 ID 检索 Qdrant
-> 取回相关文档片段
-> 拼接 RAG Prompt
-> 调用大模型
-> 流式返回答案
-> 保存聊天记录和 Token 用量
-> 前端展示答案和引用来源
```

## 4. 文档入库流程

```text
用户上传文档
-> 后端保存原始文件
-> 解析 PDF / Markdown / txt
-> 文本清洗
-> 文本切分
-> 生成 Embedding
-> 写入 Qdrant
-> 保存文档和片段元数据
```

## 5. Agent 工具调用流程

```text
用户提出任务
-> 后端把工具列表交给模型
-> 模型判断是否调用工具
-> 后端校验工具参数和权限
-> 执行业务工具
-> 工具结果返回模型
-> 模型生成最终回答
-> 记录工具调用日志
```

## 6. 企业级要求

- 所有 API 必须鉴权
- 文档和向量必须做用户或租户隔离
- 模型调用必须记录用量
- 高风险工具必须人工确认
- 上传文件必须限制大小和类型
- API Key 必须使用环境变量
