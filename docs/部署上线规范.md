# 部署上线规范

## 1. 部署目标

用 Docker Compose 一键启动：

- 前端
- 后端
- PostgreSQL
- Qdrant
- Nginx

## 2. 环境变量

必须使用 `.env`：

```text
OPENAI_API_KEY=
DATABASE_URL=
QDRANT_URL=
JWT_SECRET=
```

不要提交真实 `.env`。

只提交 `.env.example`。

## 3. 后端部署要求

- 使用生产启动命令
- 关闭 debug
- 限制 CORS 域名
- 配置日志
- 配置超时

## 4. 前端部署要求

- 执行 build
- 使用 Nginx 托管静态文件
- API 通过反向代理访问后端

## 5. 数据持久化

Docker 服务必须挂载 volume：

- PostgreSQL 数据
- Qdrant 数据
- 上传文件

## 6. 上线前检查

- [ ] API Key 不在代码中
- [ ] 数据库密码不在代码中
- [ ] CORS 不允许任意域名
- [ ] 上传文件有限制
- [ ] 用户权限已测试
- [ ] 日志不包含敏感信息
- [ ] 服务重启后数据不丢
