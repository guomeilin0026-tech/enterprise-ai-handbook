# RAG 工程规范

## 1. RAG 目标

RAG 的目标不是让模型“更聪明”，而是让模型基于企业资料回答问题。

核心要求：

- 能查到相关资料
- 回答基于资料
- 展示引用来源
- 资料不足时拒答

## 2. 文档解析

支持优先级：

```text
txt / markdown
PDF
Word
网页
扫描件 OCR
```

解析后要做：

- 去除空行
- 去除无意义页眉页脚
- 保留标题层级
- 保存来源信息

## 3. 文本切分

切分策略：

- 优先按标题和段落切
- 兜底按固定长度切
- 相邻片段可以有少量重叠

切分太长：

- 检索不精准
- Token 成本高

切分太短：

- 上下文不完整
- 模型无法理解完整语义

## 4. 检索策略

最小版本：

```text
Embedding -> 向量检索 top_k
```

进阶版本：

```text
Embedding 检索
+ 关键词检索
+ Rerank
+ 相似度阈值
```

## 5. Prompt 规范

```text
你是企业知识库助手。
请只根据资料回答问题。
如果资料中没有答案，请回答“资料中没有找到相关信息”。
不要编造资料之外的内容。
```

## 6. 引用来源

每个回答应返回：

- 文档名
- 片段编号
- 原文片段
- 相似度分数

## 7. 验收标准

- [ ] 文档能上传和解析
- [ ] 文档能切分并入库
- [ ] 问题能检索到相关片段
- [ ] 回答能展示引用来源
- [ ] 问资料外问题时能拒答
- [ ] 不同用户不能检索彼此资料
