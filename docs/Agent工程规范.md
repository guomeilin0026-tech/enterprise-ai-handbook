# Agent 工程规范

## 1. Agent 定义

Agent 是能根据用户目标选择工具并完成任务的 AI 系统。

企业 Agent 不应该无边界行动，必须在权限和工具范围内工作。

## 2. 工具设计原则

工具名要清晰：

```text
get_order_status
search_knowledge_base
create_report
```

参数要严格：

```json
{
  "order_id": "string"
}
```

返回要结构化：

```json
{
  "status": "success",
  "data": {}
}
```

## 3. 工具分类

低风险工具：

- 查询订单
- 查询知识库
- 查询用户资料

中风险工具：

- 生成报表
- 创建工单
- 发送草稿邮件

高风险工具：

- 删除数据
- 付款
- 正式发送邮件
- 提交审批

高风险工具必须人工确认。

## 4. Agent 执行流程

```text
用户输入
-> 模型判断工具
-> 后端校验权限
-> 后端执行工具
-> 保存工具日志
-> 工具结果返回模型
-> 模型输出最终回答
```

## 5. 安全要求

- 工具白名单
- 参数校验
- 用户权限校验
- 工具调用日志
- 高风险操作确认
- 失败兜底提示

## 6. 验收标准

- [ ] AI 能选择正确工具
- [ ] 工具参数能被校验
- [ ] 工具执行结果能返回给 AI
- [ ] 前端能展示工具调用过程
- [ ] 工具调用有日志
- [ ] 高风险工具不会自动执行
