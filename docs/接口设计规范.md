# 接口设计规范

## 1. 基础原则

- API 使用 REST 风格
- 返回 JSON
- 错误格式统一
- 所有业务接口默认需要鉴权
- 不在接口中返回敏感信息

## 2. 统一返回格式

成功：

```json
{
  "success": true,
  "data": {},
  "message": "ok"
}
```

失败：

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "参数错误"
  }
}
```

## 3. 核心接口

### 聊天接口

```text
POST /api/chat
```

请求：

```json
{
  "conversation_id": "string",
  "message": "string",
  "knowledge_base_id": "string"
}
```

### 文档上传接口

```text
POST /api/documents/upload
```

字段：

- knowledge_base_id
- file

### 知识库问答接口

```text
POST /api/knowledge/chat
```

请求：

```json
{
  "knowledge_base_id": "string",
  "question": "string"
}
```

返回：

```json
{
  "answer": "string",
  "sources": [
    {
      "document_name": "员工手册.pdf",
      "chunk_id": "chunk_001",
      "content": "引用片段"
    }
  ]
}
```

### 用量统计接口

```text
GET /api/usage/summary
```

返回：

```json
{
  "total_requests": 100,
  "input_tokens": 12000,
  "output_tokens": 8000
}
```

## 4. 错误码建议

```text
UNAUTHORIZED：未登录
FORBIDDEN：无权限
VALIDATION_ERROR：参数错误
MODEL_ERROR：模型调用失败
DOCUMENT_PARSE_ERROR：文档解析失败
VECTOR_SEARCH_ERROR：向量检索失败
RATE_LIMITED：请求过于频繁
```
