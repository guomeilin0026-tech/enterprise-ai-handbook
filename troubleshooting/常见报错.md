# 常见报错

## 1. `python 不是内部或外部命令`

### 现象

```text
python 不是内部或外部命令
```

### 原因

Python 没安装，或者没有加入环境变量。

### 解决

重新安装 Python，并勾选：

```text
Add Python to PATH
```

然后重新打开终端。

## 2. `ModuleNotFoundError`

### 现象

```text
ModuleNotFoundError: No module named 'fastapi'
```

### 原因

依赖没有安装，或者没有进入虚拟环境。

### 解决

进入对应后端目录：

```bash
cd apps/ai-chat-assistant/backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## 3. `uvicorn 不是内部或外部命令`

### 原因

`uvicorn` 没安装，或者虚拟环境没激活。

### 解决

```bash
pip install -r requirements.txt
python -m uvicorn main:app --reload
```

如果直接 `uvicorn` 不行，就用：

```bash
python -m uvicorn main:app --reload
```

## 4. 端口 8000 被占用

### 现象

```text
address already in use
```

### 原因

已经有另一个后端服务占用了 8000。

### 解决

换端口：

```bash
uvicorn main:app --reload --port 8001
```

同时前端里的 API 地址也要改成：

```text
http://127.0.0.1:8001
```

## 5. 前端 `npm install` 很慢或失败

### 原因

网络问题或 npm 源较慢。

### 解决

可以尝试：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

## 6. `npm 不是内部或外部命令`

### 原因

Node.js 没安装。

### 解决

安装 Node.js LTS 版本，然后重新打开终端：

```bash
node -v
npm -v
```

## 7. 前端请求后端失败

### 现象

页面显示：

```text
请求失败，请检查后端是否启动
```

### 常见原因

- 后端没启动
- 后端端口不是 8000
- 前端 API_BASE 写错
- 浏览器跨域被拦截

### 解决

先打开：

```text
http://127.0.0.1:8000/health
```

如果打不开，说明后端没启动成功。

如果后端正常，再检查前端代码里的：

```text
API_BASE
```

## 8. CORS 跨域错误

### 现象

浏览器控制台出现：

```text
CORS policy
```

### 原因

FastAPI 没允许 Vue3 的地址访问。

### 解决

后端需要配置：

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173", "http://127.0.0.1:5173"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## 9. `.env` 没生效

### 原因

常见情况：

- 没有复制 `.env.example`
- `.env` 放错目录
- 修改 `.env` 后没重启后端

### 解决

在项目目录复制：

```bash
copy .env.example .env
```

修改后重启后端。

## 10. Docker 启动失败

### 常见原因

- Docker Desktop 没启动
- 端口被占用
- `.env` 文件不存在
- 镜像构建失败

### 解决

确认 Docker 正常：

```bash
docker version
```

项目目录下执行：

```bash
copy .env.example .env
docker compose up --build
```

## 11. 文件上传失败

### 常见原因

- 文件格式不支持
- 文件太大
- 文件不是 UTF-8 编码

### 解决

教学版项目通常支持：

```text
txt
md
markdown
```

如果是 Word 或 PDF，需要后续增加解析库。

## 12. RAG 问答没有引用来源

### 原因

- 没上传文档
- 文档内容和问题不相关
- 检索没有命中相关片段

### 解决

先上传示例文档：

```text
sample-documents/员工手册.md
```

再问：

```text
年假怎么算？
```

## 13. Mock 回复不是 bug

很多项目默认使用 mock。

这是为了让你先跑通流程。

如果要接真实模型，需要修改 `.env`：

```text
USE_MOCK_LLM=false
OPENAI_API_KEY=你的 key
OPENAI_BASE_URL=你的模型接口地址
OPENAI_MODEL=模型名称
```

## 14. GitHub 上中文文件名乱码

一般 GitHub 能正常显示 UTF-8。

如果本地 PowerShell 显示乱码，可以用：

```powershell
Get-Content -Encoding UTF8 文件名.md
```
